## Setup Lokal (MAMP)

### `.env` database

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=8889
DB_DATABASE=wedding_app
DB_USERNAME=root
DB_PASSWORD=root
```

> Port `3306` jika MySQL bukan MAMP. Port `8889` adalah default MAMP.

### Perintah awal

```bash
cd /Applications/MAMP/htdocs/wedding-app

composer install
cp .env.example .env   # jika belum ada
php artisan key:generate
php artisan migrate
php artisan db:seed
php artisan storage:link
php artisan serve --host=0.0.0.0 --port=8000
```

Admin Filament: `http://127.0.0.1:8000/admin`

### Troubleshooting database

| Gejala | Solusi |
| --- | --- |
| `Connection refused` | Start MAMP, cek `DB_PORT` |
| `Unknown database` | Buat DB `wedding_app` di phpMyAdmin |
| Tabel tidak ada | `php artisan migrate` |
| Tabel kosong | `php artisan db:seed` |
| iOS data tidak sinkron | Cek `APIConfig.swift`, `php artisan route:clear` |

## iOS App

### Generate / buka project

```bash
cd ios-app/WeddingApp
xcodegen generate
open WeddingApp.xcodeproj
```

### Run

1. Backend jalan: `php artisan serve --host=0.0.0.0 --port=8000`
2. Xcode → scheme **WeddingApp** → **Run (⌘R)**

Simulator pakai `127.0.0.1:8000`. Device fisik pakai IP Mac di `APIConfig.swift`.

### Google Sign-In

Isi di `.env`:

```env
GOOGLE_IOS_CLIENT_ID=xxxxx.apps.googleusercontent.com
GOOGLE_CLIENT_ID=xxxxx.apps.googleusercontent.com
```

Isi `GIDClientID` & URL scheme di `ios-app/WeddingApp/project.yml`, lalu `xcodegen generate`.

Console: [Google Cloud Console](https://console.cloud.google.com/)

## Optimasi Performa

Jalankan setelah deploy, update config/routes, atau saat aplikasi terasa lambat di production.

### Production (disarankan)

```bash
composer install --no-dev --optimize-autoloader
composer dump-autoload -o
npm run build
php artisan migrate --force
php artisan optimize
php artisan filament:optimize
php artisan icons:cache
```

`php artisan optimize` meng-cache config, routes, views, dan events sekaligus.

### Perintah per cache

```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan event:cache
php artisan filament:optimize
php artisan filament:cache-components
php artisan icons:cache
```

### Bersihkan cache (setelah ubah `.env`, routes, atau Filament)

```bash
php artisan optimize:clear
php artisan filament:optimize-clear
php artisan cache:clear
php artisan permission:cache-reset
```

> **Lokal/dev:** hindari `config:cache` / `route:cache` saat development — bisa membuat perubahan `.env` dan route tidak langsung terbaca. Pakai `optimize:clear` jika sempat menjalankan optimize di lokal.
