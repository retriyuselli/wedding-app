### 1. Fitur Internal Chat (Vendor ↔ Admin)

**Tujuan:** Vendor bisa chat langsung ke admin/super_admin, satu thread per vendor.

**Migration baru:**

- `2026_05_16_142944_add_user_id_to_chat_sessions_table.php` — tambah kolom `user_id`

- `2026_05_17_052926_add_type_to_chat_sessions_table.php` — tambah kolom `type` ('public' | 'internal')

**Model:** `app/Models/ChatSession.php`

- Tambah `type` dan `vendor_id` ke `$fillable`

**Controller:** `app/Http/Controllers/ChatController.php`

- `vendorInternalChat()` — GET, buat/ambil 1 thread per vendor (`firstOrCreate` dengan `type=internal`)
- `vendorInternalSend()` — POST, kirim pesan dengan `sender=vendor`
- `vendorInternalPoll()` — GET, polling pesan baru (vendor side)
- `adminInternalIndex()` — GET, list semua thread internal
- `adminInternalDetail()` — GET, detail thread + pesan
- `adminInternalReply()` — POST, balas dengan `sender=admin`
- `adminInternalPoll()` — GET, polling pesan baru (admin side)

**Routes:** `routes/web.php`

```
GET  /dashboard/internal-chat              → chat.internal.vendor

GET  /dashboard/chat-internal/{token}      → chat.internal.admin.detail
POST /dashboard/chat-internal/{token}/reply → chat.internal.admin.reply
GET  /dashboard/chat-internal/{token}/poll  → chat.internal.admin.poll
```

### 3. Security Audit — Throttle Rate Limiting

**Masalah:** 11 endpoint sensitif tidak punya rate limiting (brute-force risk).

**Perbaikan** — tambah `->middleware('throttle:X,1')` di `routes/web.php`:


| Route                                              | Limit    | Alasan                            |
| -------------------------------------------------- | -------- | --------------------------------- |
| `POST /login`                                      | 10/menit | Brute-force password              |
| `POST /register`                                   | 10/menit | Spam akun massal                  |
| `POST /forgot-password`                            | 5/menit  | Email flooding                    |
| `POST /reset-password`                             | 5/menit  | Brute-force token reset           |
| `POST /dashboard/profile/password`                 | 5/menit  | Ganti password dari session aktif |
| `POST /dashboard/chat/{token}/reply`               | 60/menit | Flooding pesan admin              |
| `POST /dashboard/chat/{token}/open`                | 20/menit | Spam ubah status sesi             |
| `POST /dashboard/chat/{token}/close`               | 20/menit | Spam ubah status sesi             |
| `POST /dashboard/vendor-chat/{token}/reply`        | 60/menit | Flooding pesan vendor             |
| `POST /dashboard/vendor/payments/{payment}/verify` | 10/menit | Abuse verifikasi pembayaran       |
| `POST /logout`                                     | 20/menit | Scanner flag                      |
| `POST /kontak`                                     | 5/menit  | Spam form kontak publik           |


**Sisa 14 issues = false positive:**

- `UNVALIDATED_INPUT` di GET routes → `.all()` adalah Collection method, bukan `$request->all()`
- `XSS_BLADE_UNESCAPED` → semua `{!! !!}` dari konten admin Filament / hardcoded PHP
- `PUBLIC_WRITE` di `/login`, `/logout`, `/register`, `/kontak` → by design
