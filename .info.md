## Setup Lokal (MAMP)

### `.env` database

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=8889
DB_DATABASE=wedding_app
DB_USERNAME=root
DB_PASSWORD=root
```

> Port `3306` jika MySQL bukan MAMP. Port `8889` adalah default MAMP.

### Perintah awal

```bash
cd /Applications/MAMP/htdocs/wedding-app

composer install
cp .env.example .env   # jika belum ada
php artisan key:generate
php artisan migrate

php artisan storage:link
php artisan serve --host=0.0.0.0 --port=8000
```

Admin Filament: `http://127.0.0.1:8000/admin`

| Gejala | Solusi |
| --- | --- |

## iOS App

### Generate / buka project

```bash
cd ios-app/WeddingApp
xcodegen generate
open WeddingApp.xcodeproj
```

### Run

1. Backend jalan: `php artisan serve --host=0.0.0.0 --port=8000`
2. Xcode → scheme **WeddingApp** → **Run (⌘R)**

**API otomatis (Debug):** app memilih server yang merespons `/up` tercepat (lokal atau production).

| Lingkungan | URL lokal yang dicek |
|---|---|
| Simulator | `http://127.0.0.1:8000/api/v1` |
| Device fisik | `http://<IP-Mac>:8000/api/v1` (atur `lanHost` di `APIConfig.swift`) |

> **Device fisik:** Mac dan iPhone harus satu Wi-Fi, backend jalan (`php artisan serve --host=0.0.0.0 --port=8000`), IP Mac benar. Jika tidak, login akan gagal atau lambat karena production API (`api.weddingapp.co.id`) belum tersedia.

Paksa production di Debug: set `APIConfig.usesProductionAPI = true` (hanya jika server production sudah live).

### Google Sign-In

Isi di `.env`:

```env
GOOGLE_IOS_CLIENT_ID=xxxxx.apps.googleusercontent.com
GOOGLE_CLIENT_ID=xxxxx.apps.googleusercontent.com
```

Isi `GIDClientID` & URL scheme di `ios-app/WeddingApp/project.yml`, lalu `xcodegen generate`.

Console: [Google Cloud Console](https://console.cloud.google.com/)

### Push Notifications (APNs)

**Fungsi:** kirim alert ke iPhone (saat ini: balasan chat support → buka halaman Pesan).

#### 1. Apple Developer — buat APNs Key

1. [Apple Developer](https://developer.apple.com/account/resources/authkeys/list) → **Keys** → **+**
2. Centang **Apple Push Notifications service (APNs)**
3. Download file `AuthKey_XXXXXX.p8` (hanya sekali!)
4. Simpan ke `storage/app/apns/AuthKey.p8` (jangan di-commit)

#### 2. Konfigurasi `.env`

```env
PUSH_DRIVER=apns
APNS_KEY_ID=XXXXXXXXXX
APNS_TEAM_ID=LHH9LVRYVY
APNS_BUNDLE_ID=com.weddingapp.ios
APNS_PRIVATE_KEY=storage/app/apns/AuthKey.p8
APNS_PRODUCTION=false
```

| Variabel | Keterangan |
|---|---|
| `APNS_PRODUCTION=false` | Build Debug dari Xcode (sandbox APNs) |
| `APNS_PRODUCTION=true` | TestFlight / App Store |

Cek konfigurasi:

```bash
php artisan push:apns-status
```

#### 3. Queue worker (wajib untuk kirim push)

```bash
php artisan queue:work
# atau pakai semua service sekaligus:
composer run dev
```

#### 4. Di iPhone

1. Login ke app
2. Izinkan notifikasi saat dialog muncul (atau **More → Pengingat**)
3. Pastikan device token tersimpan:

```bash
php artisan tinker --execute 'echo App\Models\DeviceToken::count();'
```

#### 5. Uji kirim push

```bash
php artisan push:send-test ramadhona.utama@gmail.com
```

Mode tanpa APNs key (`PUSH_DRIVER=log`): push hanya muncul di `storage/logs/laravel.log`.

#### 6. Uji end-to-end (balasan support)

1. User kirim pesan dari app → **Bantuan**
2. Admin balas di Filament
3. iPhone menerima push → tap → halaman **Pesan**

> Push **tidak jalan di Simulator** — gunakan iPhone fisik.

## Optimasi Performa

Jalankan setelah deploy, update config/routes, atau saat aplikasi terasa lambat di production.

### Production (disarankan)

**Lokal** (karena server Hostinger tidak punya Node):

```bash
npm run build
git add public/build
git commit -m "Build assets"
git push origin main
```

**Server (SSH):**

```bash
composer install --no-dev --optimize-autoloader
composer dump-autoload -o
php artisan migrate --force
php artisan optimize
php artisan filament:optimize
php artisan icons:cache
```

`php artisan optimize` meng-cache config, routes, views, dan events sekaligus.

### Perintah per cache

```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan event:cache
php artisan filament:optimize
php artisan filament:cache-components
php artisan icons:cache
```

### Bersihkan cache (setelah ubah `.env`, routes, atau Filament)

```bash
php artisan optimize:clear
php artisan filament:optimize-clear
php artisan cache:clear
php artisan permission:cache-reset
```

> **Lokal/dev:** hindari `config:cache` / `route:cache` saat development — bisa membuat perubahan `.env` dan route tidak langsung terbaca. Pakai `optimize:clear` jika sempat menjalankan optimize di lokal.
